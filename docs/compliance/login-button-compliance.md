# 로그인 버튼 컴플라이언스 (카카오 · Apple) + 플랫폼 정책

W1 #4 인증의 소셜 로그인 버튼/플로우가 스토어 심사·브랜드 가이드를 통과하도록 정리한 기준 문서.
실제 SDK 연동·심사 시점에 아래 출처의 **최신본으로 재확인** 전제(정책은 갱신됨).

## 1. 플랫폼별 노출 (결정)
- **iOS**: 카카오 + **Apple로 로그인** 둘 다 노출 (Apple Guideline 4.8 충족 필수 — 아래).
- **Android**: 카카오만 노출 (Apple 로그인 숨김 — Apple 플랫폼 기능). Android 전용 레이아웃은 디자이너 제공 예정.
- 구현: 위젯에서 `defaultTargetPlatform`(테스트 override 가능)으로 분기. `dart:io Platform` 직접참조 지양(테스트·웹 고려).

## 2. Apple App Store — Guideline 4.8 (Login Services)
- 제3자/소셜 로그인(카카오)을 제공하면, **(a) 데이터 수집을 이름·이메일로 제한 (b) 이메일 비공개(릴레이) 지원 (c) 동의 없는 광고 추적 없음** 을 만족하는 로그인을 **하나 이상 함께** 제공해야 함. **Sign in with Apple**이 이를 충족.
- **카카오 단독 = 위반(리젝 위험 높음).** → iOS는 Apple 로그인 동반 필수.
- **TestFlight 비공개 베타는 4.8 강제 안 함** → W1 베타 단계는 카카오 단독으로도 통과. 단 정식 출시 전 Apple 필수.
- Android(Play)엔 동등 요건 없음.

## 3. Sign in with Apple — 버튼 HIG (필수 준수)
출처: https://developer.apple.com/design/human-interface-guidelines/sign-in-with-apple (KR: .../kr/...)
- **시스템 제공 버튼 사용**: iOS `ASAuthorizationAppleIDButton`, Flutter는 `sign_in_with_apple` 패키지의 `SignInWithAppleButton`. **커스텀 버튼 금지.**
- **승인된 문구만**: "Apple로 로그인" / "Apple로 계속하기" / "Apple로 가입하기" (앱 언어로 로컬라이즈). → 현재 디자이너 목업의 **"애플로 시작하기"는 비승인 문구 = 리젝 사유.** 교체 필요.
- 스타일: Black / White / White-outline 중 택1. 로고 비율·여백·모서리 곡률·최소 크기 임의 변경 금지.
- 다른 로그인 옵션과 **동등한 비중**으로 노출(덜 눈에 띄게 X).

## 4. 카카오 로그인 버튼 — 카카오 디자인 가이드
- 공식 **카카오 심볼(말풍선)** 포함, 배경 **`#FEE500`**(우리 `AppColors.kakao` 일치), 텍스트 색 `rgba(0,0,0,0.85)`.
- 승인 문구: "카카오 로그인" / "카카오계정으로 로그인" / "카카오로 시작하기"(허용 변형). 색·심볼 임의 변경 금지.
- 현재 목업: 색·문구 OK. **공식 카카오 심볼 포함 여부 확인 필요**(누락 시 가이드 위반).

## 5. 계정 삭제 (Apple 5.1.1(v))
- 계정 생성 앱은 **인앱 계정삭제** 제공 필수. PRD `32_계정삭제`로 충족 예정.
