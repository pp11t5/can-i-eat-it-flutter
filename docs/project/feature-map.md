## 신호등 4상태 (EatVerdict)

| 상태 | 화면 ID | 색 | 의미 |
|---|---|---|---|
| 권장 | 15 | 녹색 | 섭취 무방 |
| 주의 | 16 | 노랑 | 주의 필요, 대체 음식 제시 |
| 위험 | 17 | 빨강 | 섭취 비권장, 대체 음식 제시 |
| 확인어려움 | 18 | 회색 | LLM 신뢰도 임계 미만 폴백, 면책 고지 |

- 주의·위험 카드에는 대체 음식 1~3개 제시
- 확인어려움: 단정 표현 금지, 카피 PO 검수 필수

## 결과 카드 3섹션 (food_check F2)

1. **일반 분석** — 음식 특성·공통 정보
2. **개인화 맞춤 분석** — 사용자 트리거·알레르기 매치 결과
3. **이 음식 섭취 후 기록** — 과거 섭취 후 증상 히스토리 + 건수·평균

## 리치 푸시 in-notification 응답 (F3, P0)

- **iOS**: `UNNotificationContentExtension` + `UNNotificationAction` — 알림 내 강도·증상·완료 응답
- **Android**: `RemoteViews` + `RemoteInput` — 알림 내 강도·증상·메모 입력
- 트리거: 식후 6시간 / 매일 밤 진입점

## 앱 셸 · 라우팅 구조

- 바텀 네비: 홈 / 타임라인 / (중앙 체크 FAB) / 리포트 / 마이페이지
- go_router `StatefulShellRoute` 적용
- 스플래시 → 인증·온보딩 완료 여부에 따른 `redirect` 가드

## 온보딩 4단계 진행 인디케이터

진행 인디케이터 N/4 표시, 인트로(`04`) + 4스텝 + 완료(`09`)

| 스텝 | 화면 | 내용 |
|---|---|---|
| 1/4 | 05_질환선택 | GERD 고정, 다중 질환 확장 가능한 UI 구조 |
| 2/4 | 06_증상빈도 | "최근 4주간…" 복수선택 + 진단여부 (심각도 통합, D1) |
| 3/4 | 07_트리거음식 | 일반 트리거 리스트 + 기타 텍스트 (선택) |
| 4/4 | 08_알레르기복용약 | 리스트 선택 (선택) |

완료 후 `POST /v1/users/profile` 일괄 전송 → 홈 진입.

## 주간 리포트 차트 3종 (report F4, MVP)

1. **신호등 분포 도넛** — 권장/주의/위험 끼수·비율
2. **증상 발생 시간대 막대** — 시간대별 분포
3. **트리거 후보 음식** — "○○ 후 증상 발생률이 평균보다 높아요"

패키지: `fl_chart` (차트), `share_plus` (OS 공유시트)
