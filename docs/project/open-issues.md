### OI-01 강제 업데이트 vs API 버전 분기

- **배경**: PRD v1에 언급 없음. 서버 API가 버전 분기를 지원하지 않을 경우 구버전 클라이언트 강제 업데이트가 필요.
- **선택지**:
  - A. 서버 응답에 `min_client_version` 필드 → 클라이언트가 비교 후 강제 업데이트 팝업
  - B. 스토어 In-App Update API (Android) / StoreKit (iOS)
  - C. `/v2/` URL prefix 버전 분기 (API 유지)
- **현재 상태**: 비차단. W4~W5 릴리즈 준비 시 PO + BE와 결정.
- **결정되면**: ADR 신규 작성 + `api-contract.md` 갱신.
