## 피벗 서사

원래 루틴 앱으로 시작했다가 **GERD(역류성 식도염) 식이앱으로 피벗**됐다. 피벗 이후 5주 MVP 일정으로 확정됐으며, 역할별 간트 로드맵이 존재한다.

## 문제 정의

- GERD 환자는 어떤 음식이 자신의 증상을 유발하는지 즉석에서 판단하기 어렵다.
- 식사–증상 인과를 기억에 의존 → 증상이 반복돼도 무엇을 줄여야 할지 모른 채 방치된다.

## 핵심 가치

1. **식사 전 신호등 판정** — 음식 검색 → 권장/주의/위험 + 개인화 사유로 즉각 의사결정 보조
2. **식사–증상 누적 데이터** — 서비스의 핵심 자산
3. **주간 리포트** — 신호등 분포·증상 시간대·트리거 후보로 자기 패턴 파악, 의료진과 공유

## 타겟 사용자

- **주 타겟**: 20~30대, 병원 갈 정도는 아닌데 증상 해결은 하고 싶은 미진단군
- **1차 페르소나**: "무엇을 먹어야 안전한지 매 끼니 고민하는 만성 GERD 환자"
- **국내 타깃** (카카오 단독 인증 근거)

## MVP 범위 (PRD v1 확정)

**포함 (W1~W5, 클로즈드 베타 50명)**:
- F0 인증: 카카오 소셜 로그인 (Mock 선개발 후 교체)
- F1 온보딩: 4단계 입력 (질환선택·증상빈도·트리거·알레르기약)
- F2 신호등 판정: 텍스트 검색 단독, 결과 4상태, 카드 3섹션, 음식 직접추가
- F3 식사·증상 기록: 리치 푸시 in-notification 응답, 타임라인
- F4 in-app: 주간 리포트(차트 3종) + 마이페이지 요약 + OS 공유시트
- 공통: 계측 P0, 디자인 시스템, 알림 설정, 보안

**베타 이후 (W6+)**:
- 서버사이드 PDF 생성
- 트리거 추론 배치 고도화 (2주↑ 데이터 임계)
- 주간 자동 푸시
- 인앱결제 (월 ₩9,900)

**P2 확장 (기간 미정)**:
- F2 카메라 촬영·메뉴판 스캔
- 다중 질환 지원
- 커뮤니티 / 건강검진 OCR / 마트 바코드 스캔

## 의료 규제 가드레일

- "치료·진단·처방" 표현 금지
- "주의/권장" 톤 유지 + 면책 고지 ("본 앱은 진단·치료 서비스가 아닙니다")
- 모든 결과 카피 PO 출시 전 검수 필수
- `MedicalDisclaimer` 위젯: 전 결과 화면에 노출
