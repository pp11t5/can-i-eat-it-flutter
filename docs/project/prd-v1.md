## 1. 제품 개요

### 1.1 한 줄 정의

역류성 식도염(GERD) 증상이 있지만 **아직 병원에 갈 정도는 아니라고 느끼는 사람**이, **식사 전 음식을 검색해 신호등으로 판정**받고 식사–증상 데이터를 누적해 **주간 리포트로 자신의 패턴을 확인·공유**할 수 있는 모바일 앱.

### 1.2 문제 정의

- GERD 환자는 어떤 음식이 자신의 증상을 유발하는지 **`즉석에서 판단하기 어렵다`**.
- 식사–증상 인과를 기억에 의존 → 증상이 반복돼도 무엇을 줄여야 할지 모른 채 방치된다.

### 1.3 핵심 가치

1. **식사 전 신호등 판정** — 음식 검색 → 권장/주의/위험 + 개인화 사유로 즉각적 의사결정 보조
2. **식사–증상 누적 데이터** — 서비스의 핵심 자산
3. **주간 리포트** — 신호등 분포·증상 시간대·트리거 후보로 **`자기 패턴 파악`**, 의료진과 공유

### 1.4 타깃 사용자

- 20~30대, 병원갈 정도는 아닌데, 증상 해결은 하고 싶은 미진단군.
- 1차 페르소나: "무엇을 먹어야 안전한지 매 끼니 고민하는 만성 GERD 환자"

### 1.5 비범위 (Out of Scope, v1)

- **F2 카메라 촬영·메뉴판 스캔** (D4) → P2 확장
- **서버사이드 PDF 생성 / 주간 자동 푸시** → 베타 이후
- 다중 질환 / 커뮤니티 / 건강검진 OCR / 마트 바코드 스캔 → P2 확장
- 인앱결제 → 베타 이후(W6+)

## 3. 기능 요구사항 (MVP)

### 진입 · 인증 플로우

`01_스플래시` → `02_로그인`(카카오) → `03_약관동의` → `04_온보딩_인트로` → 온보딩 4스텝 → `09_온보딩_완료` → `10_홈`

### F1. 온보딩 — 질환·건강 정보 등록 [P0]

**목적**: 개인화 판정 룰셋 세팅 (베이스라인 확보)

**스텝 구성 (인트로 + 4스텝, 진행 인디케이터 N/4)**

1. **질환 선택**: GERD 고정, 다중 질환 확장 가능한 UI 구조
2. **증상 빈도 + 심각도**: "최근 4주간 어떤 증상이 얼마나 자주 있었나요?" 복수선택 UX라이팅
    - 예: "주에 1번 이상 속이 쓰리거나 신물이 올라와요" / "밥을 먹고 나면 기침이 나요" / "입에서 신맛·악취가 느껴져요" / "누우면 가슴이 답답해져요" / "목에 이물감이 있어요"
    - 진단 여부: "예전에 진단받았지만 지금은 관리만 하고 있어요" (단일 선택)
    - 하단 면책 고지 / 룰셋이 `상·중·하` 자동 라벨링 (별도 심각도 스텝 없음 — D1)
3. **트리거 음식** (선택): 일반 트리거(매운 음식·카페인·튀김 등) + 기타 텍스트
4. **알레르기·복용약** (선택): 리스트 선택
- **완료**: 완료 카드 → 홈 진입

**담당별 산출**

- Designer: 인트로 + 4스텝, 진행 인디케이터, 다음/건너뛰기, 면책 고지, 완료 카드
- Flutter: 로컬 캐싱 후 일괄 전송
- Backend: `POST /v1/users/profile`, `PUT /v1/users/profile` / 스키마 `users`, `user_conditions`, `user_medications`, `user_triggers`

### F2. 식사 전 메뉴 체크 — 신호등 판정 [P0, 핵심]

**목적**: "이거 먹어도 되나?"에 즉시 답한다. **(D4: 텍스트 검색 단독, 카메라·메뉴판 스캔 보류)**

**플로우**: 홈/검색바 → `13a 검색_입력전` → `13b 검색_입력후`(자동완성·결과 리스트) → `14 로딩중` → 결과 카드(`15 권장`/`16 주의`/`17 위험`/`18 확인어려움`)

**상세 요구사항**

- **검색**: 텍스트 입력 → "검색 결과 N건" 리스트, 각 항목에 분류·정보(예: "음료 · 카페인 약 150mg")
- **음식 미존재 시**: "찾는 음식이 없나요?" → **사용자 직접 추가**(카테고리 자동 분류, 승인 시 다른 사용자에게도 노출) 후 기록
- **결과 카드 구조** (권장 기준):
    - 신호등(🟢권장/🟡주의/🔴위험) + 음식명 + `약 480 kcal · 한식` + 분류 태그
    - **일반 분석(공통 정보)**: 음식 특성 설명
    - **개인화 맞춤 분석(회원님 기반)**: "트리거 음식 매치 없음" / "알레르기 해당 없음" 등 등록 정보 대조
    - **이 음식 섭취 후 기록(N건)**: 과거 섭취 후 증상 히스토리 + 평균(예: "최근 3건 평균: 편안") + 모두 보기
    - 하단 CTA: **[다시 검색] / [지금 먹었어요]**(→ F3 식사 기록)
    - 🟡주의·🔴위험 카드에는 **대체 음식 1~3개** 제시
- **확인 어려움**: 신뢰도 임계 미만/매칭 실패 시 "확인이 어려워요" + 면책 고지 (단정 금지)

**담당별 산출**

- Designer: 검색 입력전/후, 로딩, 결과 4종, 음식 직접 추가
- Flutter: 검색 자동완성, 결과 캐시 — ※ image_picker·카메라 권한 플로우 제외 (D4)
- Backend: `POST /v1/foods/analyze` (**text**) → `{label, confidence, signal, reason_general, reason_personal, kcal, category, alternatives[], history_summary}` · `POST /v1/foods`(사용자 음식 추가, 승인 큐)
    - 파이프라인: LLM(Gemini Flash/GPT-4o-mini) + 식품영양성분 DB 매칭 + GERD 룰셋 후처리 + Redis 캐시 + 트리거/알레르기 개인화 매칭
- 데이터 모델: `foods`, `food_alternatives`, `user_food_submissions`, `analysis_logs`

**비기능**: P95 응답 ≤ 3초 / 검색 자동완성 캐시 활용

### F3. 식사 후 기록 + 증상 추적 [P0]

**목적**: 음식–증상 누적 데이터 확보 (서비스 핵심 자산)

**플로우**

- F2 결과에서 "지금 먹었어요" 또는 홈에서 식사 기록
- **식후 6시간** 푸시 → "지금 속은 어때요? 6시간 전 OOO을 드셨어요" → 증상 기록
- **매일 밤** 특이 이벤트 입력 진입점 푸시 / 마이페이지에서 상시 입력

**증상 기록**

- 상태: **편안 / 괜찮 / 보통 / 불편 / 심함** (5단계 이모지, 강도 0~5 대응)
- **증상 종류(복수 선택)**: 속쓰림 / 신물 / 기침 / 목 이물감 / 가슴 답답
- **발생 시간** 편집 / **연결된 식사** 편집(예 "13:24 된장찌개 외 4개")
- 메모(자유 입력) / 기록 삭제 · 식사 상세: 섭취 음식 + 신호등

**타임라인**: 무한 스크롤, 일자/끼니 그룹

**담당별 산출**

- Designer: 증상 기록·수정, 발생시간/연결식사 수정, 타임라인, 식사 상세, iOS/Android 리치 푸시
- Flutter: FCM 핸들러, iOS/Android 리치 알림 응답, 타임라인 무한스크롤, 오프라인 큐
- Backend: `POST /v1/meals`, `POST /v1/symptoms`, `PATCH /v1/symptoms/{id}`, `GET /v1/meals?from=&to=` / S3 / FCM 워커(Cron)
- 데이터 모델: `meals(...)`, `symptoms(symptom_id, meal_id, severity, types[], memo, occurred_at, recorded_at)`
- **공수**: FE 7d / BE 8d / DE 3d

### F4. 마이페이지 · 주간 리포트 · 알림 설정 [P0/P1] (D3 개정)

**MVP 포함**: 마이페이지, 이번 주 요약, 주간 리포트(차트 3종), OS 공유시트

**베타 이후**: 서버사이드 PDF 생성, 트리거 추론 배치 고도화(2주↑ 임계), 주간 자동 푸시, 인앱결제

**마이페이지** 

- 프로필(이름, "GERD · 관리 N일째", 편집 → `31` 프로필·건강정보, `35` 알레르기·복용약)
- **이번 주 요약 카드**: 기간, 식사 횟수, 증상 수, 신호등 분포(🟢N 🟡N 🔴N), 상세보기 → 주간 리포트
- 설정: 알림 설정(`37`, OS 차단 상태 `37b`) / 약관(`34`) / 계정: 로그아웃(`31a`), 계정 삭제(`32`+`32a`, 유예 `02a`)

**주간 리포트 (`36`)**

- 주차/기간 헤더 + 우상단 **공유**(→ `37/38` iOS/Android OS 공유시트)
- **신호등 분포**(도넛): 권장/주의/위험 끼수·비율
- **증상 발생 시간대**(막대): 시간대별 분포
- **트리거 후보 음식**: "○○ 후 증상 발생률이 평균보다 높아요" (예: 커피·카페인 4/5회 80%)
- 프레이밍: "사용자가 자기 데이터를 의료진과 공유" (앱이 진단한다는 표현 금지)

**담당별 산출**

- Designer: 마이페이지, 주간 리포트(차트 3종), 알림설정, 편집/계정 화면, 공유시트
- Flutter: fl_chart, share_plus(OS 공유시트), 리포트 캐싱
- Backend: `GET /v1/reports/weekly?week=YYYY-Www`, `GET /v1/users/me/summary` / 주간 집계 / (PDF·트리거 추론 배치는 베타 이후)
- 데이터 모델: `weekly_reports`, `trigger_candidates`
- **공수**: FE 6d / BE 5d / DE 4d *(MVP는 in-app 리포트+OS 공유, 서버 PDF 제외)*

- **인증·계정 [P0]**: 카카오 로그인 + 약관·동의 이력 + 로그아웃 + 계정 삭제(유예)
